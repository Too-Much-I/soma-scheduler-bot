# Soma Scheduler Bot: 소마 멘토링/특강 노션 자동 등록기

소마 멘토링 및 특강 신청 시, 일일이 팀 노션 페이지에 등록해야 하는 번거로움을 해결하기 위한 자동화 도구입니다. 이 프로젝트는 크롬 확장 프로그램(프론트엔드)과 API 통신을 담당하는 Node.js 서버(백엔드)로 구성되어 있습니다.

## 필수 준비 사항

이 시스템을 온전히 구축하고 실행하려면 다음 세 가지가 사전에 준비되어야 합니다.
1. **Notion API Secret Key**: 노션 개발자 센터에서 통합(Integration)을 생성하여 발급받은 키
2. **Notion Database ID**: 멘토링 일정을 저장할 팀 노션 데이터베이스의 고유 ID
3. **서버 호스팅 환경**: 백엔드 코드를 상시 실행해 둘 공간 (현재 가이드에서는 Replit 기준)

---

## 1. 백엔드(서버) 설정 및 실행 방법

백엔드 서버는 노션 API와 직접 통신하여 데이터를 기록 및 삭제하는 역할을 합니다.

### 환경 변수(.env) 설정
서버를 실행하기 전, 서버 환경(Replit의 Secrets 탭 또는 로컬의 `.env` 파일)에 아래 두 가지 환경 변수를 반드시 등록해야 합니다.
* `NOTION_SECRET` : 발급받은 노션 API 키
* `NOTION_DATABASE_ID` : 연결할 노션 데이터베이스 ID

### 서버 실행 (Replit 기준)
1. 백엔드 폴더의 코드를 Replit 서버에 업로드합니다.
2. [Run] 버튼을 눌러 패키지를 설치하고 서버를 가동합니다.
3. 정상적으로 실행되면 화면에 부여되는 **백엔드 서버 URL**(예: `https://...replit.dev`)을 복사해 둡니다.

---

## 2. 프론트엔드(확장 프로그램) 설정 및 설치

확장 프로그램은 웹페이지의 버튼 클릭을 감지하고 데이터를 백엔드 서버로 전송합니다.

### 서버 주소 연결
1. 다운로드한 코드 중 `frontend/content.js` 파일을 엽니다.
2. 코드 내의 `fetch` 함수 목적지 URL을 위에서 띄운 **본인의 백엔드 서버 URL**로 변경하고 저장합니다.

### 크롬 브라우저 설치
1. 크롬 브라우저에서 `chrome://extensions/` 페이지로 이동합니다.
2. 우측 상단의 [개발자 모드]를 활성화합니다.
3. 좌측 상단의 [압축해제된 확장 프로그램을 로드합니다]를 클릭하여 `frontend` 폴더를 통째로 선택합니다.

---

## 3. 사용 방법

1. 소마 마이페이지의 멘토링/특강 신청 페이지로 이동합니다.
2. 원하는 항목의 [신청하기] 또는 [취소하기] 버튼을 클릭합니다.
3. 브라우저 상단에 "멘토링 일정이 노션 달력에 등록/삭제되었습니다!"라는 알림창이 뜨면 정상 통신된 것입니다.
4. 팀 노션 페이지를 새로고침하여 데이터가 연동되었는지 확인합니다.

---

## Troubleshooting (안 될 때 확인하세요)

| 증상 | 확인 사항 |
| :--- | :--- |
| **반응이 없음** | 개발자 도구(F12) 콘솔에서 "소마 확장 프로그램 content.js 정상 주입 완료!"가 찍히는지 확인하세요. |
| **Failed to fetch 오류** | 백엔드 서버(Replit)가 켜져 있는지 확인하세요. 무료 호스팅의 경우 일정 시간 후 수면(Sleep) 상태로 진입하므로, 다시 Run 버튼을 눌러 깨워야 합니다. |
| **노션에 안 보임** | 노션 페이지 상단의 [필터]를 '없음'으로 설정했는지, 데이터베이스 속성(이름, 일시 등)이 코드와 정확히 일치하는지 확인하세요. |
| **빨간 에러 로그** | 광고 차단기(AdBlock)로 인한 흔한 에러일 수 있으니, 알림창이 정상적으로 뜬다면 무시해도 됩니다. |
